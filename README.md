# Pythonで機械学習
JAVAで作成した[戦車ゲーム](https://github.com/LineTeo/java-practice2026)をPythonのOptuneを使った機械学習で強化する演習
演習で学んだメモ等を含めてまとめている
### 学習の3要素

| Optunaの用語 | 意味 | コード上の箇所 |
| :--- | :--- | :--- |
| **Trial (試行)** | ツマミを「一度」設定して試してみること。 | `objective(trial)` 関数 |
| **Study (調査)** | 試行を何度も繰り返し、最適解を探す「プロジェクト全体」。 | `optuna.create_study(...)` |
| **Sampler (サンプラー)** | 次にどの数値を試すか決める「脳」。今回は **CMA-ES**。 | `CmaEsSampler(...)` |
| **Objective Function** | Javaを動かして「勝率（スコア）」を返す評価関数。 | `return win_rate` |
---

## 強化目標
- JAVAで作成した戦車ゲームはプレーヤーが動かす戦車1台（味方）に対してPC側が動かす戦車（敵）は3台
- 味方戦車は火力、防御力ともに敵戦車より大きく、特に正面防御が強力。
- 双方を同じアルゴリズムでPC同士で戦わせたときは味方戦車の圧勝（勝率90%以上）
- ここで敵側のアルゴリズムだけを強化して、敵側勝率を上げることが機械学習の目的
---

## 1. JAVA側の準備
- 敵戦車の戦術アルゴリズムを行動を数値で判断するように構築
- パラメータを引数で指定されたJSON形式のデータファイルで受け取れるようにする
- プレイヤー側も敵側と同じアルゴリズムでPCが操作するが、強化は敵側だけなのでパラメータは分けておく
- 学習高速化のため、グラフィック（GUI）を使わずに戦いをおこなうソフトを準備
---

## 2. `optimize_ai.py`：学習のメイン処理
AIをつかって作成したコードを見ながら、主にOptuna周りについて、コードの意味や使い方などをAIと対話しながら学習と修正

### ① パラメータの提案 (`suggest_float`)
```python
zone1 = trial.suggest_float("THREAT_ZONE_1", 0.1, 0.5)
```
* **意味:** 「`THREAT_ZONE_1` という名前で、0.1から0.5の間で良さそうな数値を一つ選んでくれ」とOptunaに依頼する
* **学習の仕組み:** 最初はランダムに近いが、回数を重ねると、Optuna（CMA-ES）は「0.4付近に設定した時の方が勝率が良いな」と学習し、その付近を集中的に提案するようになる。

### ② 学習戦略の設定 (`CmaEsSampler`)
```python
sampler = CmaEsSampler(
    n_startup_trials=20,    # 最初の20回はあえてランダムに動く
    restart_strategy="ipop", # 行き詰まったら探索範囲を広げてやり直す
)
```
### ③ データの永続化 (`storage`)
```python
storage=STUDY_DB_PATH  # sqlite:///optuna_study.db
```
* **意味:** 試行結果をSQLite（軽量なデータベース）に保存
* **メリット:** これにより、途中でプログラムを止めても、**「前回の続きから学習を再開」**することが可能になる。
---

## 3. `analyze_results.py`：結果の可視化と分析
学習が終わった後、「結局どのツマミが重要だったのか？」を分析するTOOL。
目的から方法まで含めAIが作成したもの

### ① パラメータ重要度 (`get_param_importances`)
```python
importance = optuna.importance.get_param_importances(study)
```
* **ここが肝:** 13個もパラメータがあると、「実はこの3つさえ調整すれば勝率が決まり、残りの10個はあまり関係なかった」ということがある
* これをグラフ化することで、**「どの戦術閾値が勝敗を分ける決定打なのか」**を客観的に知ることができる

### ② 分布の確認 (`scatter`)
コード内の `ax.scatter(...)` の部分は、2つのパラメータ（例：Zone1とZone2）を縦軸・横軸にとって、勝率を「色」で表現
* 濃い緑の点（高勝率）が集まっている場所があれば、そこが**「戦術的なスイートスポット」**であると視覚的にわかる。
---

## 4. 全体のワークフロー
1. [Python]** `CmaEsSampler` が「次はこれで行こう」と13個の数値を決定。
2. [Python]** 数値をJSONに書き出し、Javaを `subprocess` で叩く。
3. [Java]** JSONを読み込み、戦術クラスに反映して5000試合実行。勝率を出力。
4. [Python]** Javaの出力を読み取り、Optunaに「この数値は勝率0.xxだったよ」と報告。
5. [Optuna]** データベースに記録し、CMA-ESの数式を更新して **Step 1** に戻る。
このループを最終的には1000回まわして学習

### 感想
強化学習を行った結果、目標を達成したことには満足したが、一方で学習した条件で勝率を上げることはできても、同じパラメータでユニットの初期配置や動かす順番を変えただけで勝率が大きく変化することも分かった。
また、実際に強化アルゴリズムを相手にプレーしてみても、あまり強くなった印象はない。学習の方法は学べたが、強化学習で強いAIを作るにはより高度な戦術や全体を見た戦略を立てるアルゴリズムが必要そうだ
